S-02 teach-mode 練習場

実サイトを使う前に、この場で「クリックして教える」体験を 1 回済ませて成功体験で終わらせる。ここで教えた対象は拡張の storage に保存されるので、次回このページを開いても数字が読み取れる = 拡張がちゃんと動いていることの証明にもなる。

拡張が入っていないと動きません: この練習場は many-ai-usage 拡張の teach-mode を起動して初めて機能します。まだの方はストアからどうぞ。

手順
  1. ブラウザ右上の many-ai-usage アイコン をクリック → 「このページを教える」
  2. ページ上の数字（例: 62 %）にマウスを重ねると候補が薄オレンジで囲まれる
  3. クリックして確定 → tooltip で「62 %」と読み取ったかプレビューが出る
  4. ラベル（"Model A weekly" など）を付けて保存 → ダッシュボードに反映

S-03 実サイト向けレシピ集

各 AI サービスの usage ページへの辿り方と、「どの数字を教えればいい？」の言葉ガイド。実 URL・CSS selector・スクリーンショットは載せない（サービス側の変更で腐る／同梱すると "公式サポート" と誤解される／商標・ToS リスク）。あくまで人間が読んで自力で辿れる粒度で。

サービス usage ページの辿り方 教える対象（言葉で）
Claude
Web / Claude Code CLI		 claude.ai にログイン → 設定 → Usage（Plan & billing） 「Weekly usage」の ％表示、および「Resets on ○月○日」の日時。Claude Code CLI と Web は同じサブスク枠を共有する（CLI 側に単独 usage ページは無い）。
Codex CLI
ChatGPT サブスクを共有		 chatgpt.com にログイン → 設定 → Manage subscription / Usage Codex CLI は ChatGPT Plus / Pro / Team の枠をCLI から消費する。教える対象は Web ChatGPT の usage 側にある「5-hour window」の残数（例: 18 / 40）や、モデル別残メッセージ数。
Grok
xAI		 grok.com にログイン → 設定 / Subscription 今週の残クレジット・残メッセージ数、または reset 日時。Premium / SuperGrok でプラン差あり。
GitHub Copilot github.com → Settings → Copilot（Individual プランは個人設定側、Business は organization 側） 「Premium requests」の使用量 / 上限、reset 日。
Cursor cursor.com にログイン → Settings → Usage Fast requests の残数、Slow への切替タイミング。Pro / Business でプラン差あり。

対象外 (API 従量課金): Anthropic Console API・OpenAI Platform API・xAI API 等の API キー方式（従量課金）は残高管理の対象外です。本拡張は サブスクリプションの残枠管理に絞っています。

このセクションで達成したいこと: 「うちの契約プランではどこ？」を各サービスで自力で辿れるようにする。表は最終版でも「最終確認日: YYYY-MM-DD」バッジ付きにして腐り度を可視化する。

S-04 よくある質問 / 落とし穴

Q. usage の数字は外部（AI）に送られますか？
A. いいえ。解析はブラウザ内のみで、ページ HTML を外部 AI に送りません。サーバー・アカウント・クラウド同期もありません。
Q. 別 PC でも同じ設定を使いたい
A. 現時点では同期しません（クラウド同期を持たないポリシー）。設定のエクスポート / インポート機能で移してください。
Q. サービス側の DOM が変わって読めなくなった
A. 該当タイルの「再学習 (Re-teach)」から、新しい位置をもう一度クリックしてください。旧 fingerprint は自動失効します。
Q. ログインしていない状態で読めますか？
A. いいえ。各サービスの usage ページは要ログインです。この拡張はあなたの既存ブラウザセッションを読むだけで、代理ログインはしません。
Q. API キー方式（従量課金）の残高は？
A. 現時点では対象外です。サブスクの残枠管理に焦点を絞っています。
Q. うまく動かない
A. GitHub Issue で報告してください（サービス名 / プラン種別 / スクショ）。selector 修正はユーザー側で完結する設計なので、多くは Re-teach で解決します。
このセクションで達成したいこと: 不安（プライバシー・壊れた時・別 PC）に先回りで答える。同時に「同梱プロバイダを持たない理由」を自然に伝える。
