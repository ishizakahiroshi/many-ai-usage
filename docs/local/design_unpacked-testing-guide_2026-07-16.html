対象: many-ai-usage v0.1.0（Chrome / Firefox 拡張） unpacked テスト 2026-07-16 (木) 作成

v0.1.0 unpacked 実機テストガイド

3 分でわかる TL;DR
ローカルビルドした dist/chrome と dist/firefox を各ブラウザに「開発者モードで」読み込む。テストは 5 シナリオ（空状態 / Try samples / 冪等性 / teach-mode / 使い方ページ遷移）。teach-mode は専用の新規タブと Continuous mode を使い、複数 metric とキャンセル時の全破棄まで確認する。

1. 前提の確認

テストに入る前に以下 3 点を確認する。すべて既に済んでいるはずだが、環境が変わっているときは念のため。

確認事項確認方法状態
ビルド成果物の存在C:\dev\github\public\many-ai-usage\dist\chrome と dist\firefox に manifest.json があること。無ければ pnpm run build:chrome / pnpm run build:firefox を実行✓
typecheck / testpnpm run typecheck と pnpm test が緑（24 tests / 6 test files）✓
Registry の公開curl -I https://raw.githubusercontent.com/ishizakahiroshi/many-ai-cli/main/resources/usage-links/providers.json が HTTP/2 200✓

全て ✓ になっていれば、拡張は Try samples の Remote Registry fetch を含めて完全に動く状態。

2. Chrome への unpacked インストール

  1. アドレスバーに chrome://extensions/ と入れて開く
  2. 右上の「デベロッパー モード」トグルを ON（青くなる）
  3. 左上に現れる「パッケージ化されていない拡張機能を読み込む」を押す
  4. ファイル選択ダイアログで C:\dev\github\public\many-ai-usage\dist\chrome フォルダに入って、そのまま「フォルダーの選択」ボタンを押す
    迷いポイント: ここは「フォルダの中に入って、フォルダ自体を選ぶ」動作。
    ❌ リポルート C:\dev\github\public\many-ai-usage を選ぶ → manifest 見つからず失敗
    ❌ dist\chrome の中の manifest.json を選ぶ（Firefox の作法と混同）→ ダイアログが閉じない
    ✅ dist\chrome フォルダを選択状態で「フォルダーの選択」
  5. 拡張カードが出て「many-ai-usage v0.1.0」が表示される。右上のパズルアイコン（拡張メニュー）に M か many-ai-usage のアイコンが並ぶ
  6. 拡張アイコンを右クリック →「拡張機能を固定」でツールバーに常駐させると以降が楽

3. Firefox への unpacked インストール

  1. アドレスバーに about:debugging#/runtime/this-firefox と入れて開く
  2. 「一時的なアドオンを読み込む…」ボタンを押す
  3. ファイル選択ダイアログで C:\dev\github\public\many-ai-usage\dist\firefox\manifest.json を選ぶ（フォルダではなくファイルを選ぶ）
    Chrome との違い: Firefox は manifest.json ファイルそのものを指定する。
  4. 拡張カードが出て「many-ai-usage 0.1.0」が表示される
注意: Firefox の「一時的なアドオン」はブラウザを閉じると消える。テストごとに再読込が要る。恒常的に使うなら AMO 提出後の署名付きパッケージを待つこと。

4. テストシナリオ 5 種

順番通り実行するのが推奨。T1 が終わってから T2 → T3 → T4 → T5。各シナリオの「成功基準」を満たしたら次へ。

T1空状態フロー

ねらい: 初回起動時に「Try samples」ボタンが正しく出るか
  1. 拡張アイコンをクリックして popup を開く
  2. popup 全体を確認
providers が既に登録済みなら: 別セッションのテストで残っている可能性。chrome://extensions/ → many-ai-usage カードの「詳細」→ 一番下の「拡張機能のデータを消去」を押してから拡張アイコンを再クリック
成功基準:
  • ヘッダ「many-ai-usage v0.1.0」
  • 大きな文字で「No providers yet」
  • その下に「Try six URL-only samples, or add your own usage page.」
  • 2 つのボタン: 「Try samples ▸」 と 「使い方を見る →」

T2Try samples フロー（Remote Registry fetch）

ねらい: 明示同意 → GitHub raw から 6 サービス取得 → storage に反映
  1. popup の「Try samples ▸」を押す
  2. options タブが新規に開く（options.html?trySamples=1）
  3. options 画面中央に確認ダイアログが出る
  4. ダイアログの内容を確認: fetch URL が https://raw.githubusercontent.com/ishizakahiroshi/many-ai-cli/main/resources/usage-links/providers.json であること + 「取得するのは表示名と URL パターンだけ」の説明
  5. 「取得する」ボタンを押す
  6. 数秒待つ（ネットワーク遅延次第）
成功基準:
  • ダイアログが閉じる
  • options の左サイドバーに 6 タイル: Claude / Codex CLI / Grok / GitHub Copilot / Cursor / Ollama
  • 各タイルは「Needs access」or「Not captured」表示（selector 未教示状態）
  • トースト:「Added 6 sample providers. Open each usage page and teach the visible value.」
失敗パターン: ダイアログにエラー「Sample registry request failed (404)」→ providers.json が main に merge されていない。gh pr list --repo ishizakahiroshi/many-ai-cli --state merged で確認

T3Try samples の冪等性

ねらい: 2 回目の Try samples でも既存 provider を上書きせず、重複追加もしないこと
  1. T2 が完了した状態のまま、options の左サイドバー最上部にある「Need a starting point?」ブロックに注目
    Try samples の 6 個が既に登録されているので、通常この onboarding ブロックは非表示になっているはず
  2. もし onboarding ブロックが消えていることを確認したら、URL 直打ちで chrome-extension://<id>/options.html?trySamples=1 を開いて強制的にダイアログを出す
    <id> は chrome://extensions/ の many-ai-usage カードで確認できる
  3. 「取得する」を押す
成功基準:
  • タイル数が 6 のまま増えない
  • トースト:「Sample providers are already registered. Existing settings were not changed.」
  • 既存タイルの teach 済み設定（もしあれば）が保たれている

T4teach-mode Continuous flow

ねらい: 専用の新規タブで複数 metric を連続 teach し、完了時にまとめて保存されることを確認
  1. https://chatgpt.com/codex/cloud/settings/analytics#usage を同じウィンドウで 2 タブ開き、ChatGPT にログインしておく
  2. 拡張の options タブに戻る（chrome://extensions/ → many-ai-usage →「オプション」or popup の ⚙ options から）
  3. 左サイドバーで Codex CLI を選択
  4. 右パネル下部の 「＋ Track this element」 ボタンを押す
  5. Chrome が Codex usage を3 個目の新規タブとして開き、そのタブだけに picker を表示する
  6. ページ上の数値要素にマウスを重ねると薄オレンジのハイライトが出る
  7. 「X% 残り」の数字部分をクリックし、右上 panel の Saved が 1 になる
    日付の年号へマウスを重ねても「2026%」にならず、リセット文脈の数字は候補にならないことを確認する
  8. 続けて「残りのクレジット」をクリックし、Saved が 2 になる
  9. panel の Rename で片方の名前を変更し、一覧へ反映されることを確認する
  10. Done and return を押し、新規 picker タブが閉じて options タブへ戻る
  11. Codex CLI の「Tracked elements」に taught metric が 2 つ表示される
成功基準:
  • 既存 2 タブではなく、新規の 3 個目のタブだけに picker が出る
  • percentage と credits の 2 metric を 1 セッションで保存できる
  • tooltip が年号を usage 値として表示しない
  • options の「Tracked elements」と popup に 2 metric が反映される
キャンセル確認: もう一度 Track this element を開始して 1 metric を選び、Esc を押す。options に戻った後、Tracked elements が増えていなければ全破棄も成功。

T5使い方ページ遷移

ねらい: popup / options の「使い方を見る →」リンクが正しく github.io に飛ぶ
  1. popup を開く（provider 空状態でも、6 個登録済み状態でも良い）
  2. 空状態なら「使い方を見る →」ボタン、登録済みなら options の「Try samples」onboarding 内のリンクを押す
成功基準:
  • 新規タブが開いて https://ishizakahiroshi.github.io/articles/many-ai-usage/usage.html が表示される
  • ページが 404 でない
  • 使い方ページの各セクションが崩れずに表示される

6. 不具合が出た時のトラブルシュート

拡張の Service Worker のログを見る（Chrome）

  1. chrome://extensions/ を開く
  2. many-ai-usage カードの中で「Service worker」というリンク（青字）をクリック
  3. DevTools が開くので Console タブでエラーを確認
Service worker が非アクティブと表示される場合、拡張アイコンを 1 度クリックしてから再度リンクを探すと表示される

Firefox 側の Console

  1. about:debugging#/runtime/this-firefox を開く
  2. many-ai-usage の「調査」ボタン
  3. DevTools の Console タブでエラーを確認
