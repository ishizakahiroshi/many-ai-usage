v0.1.0 unpacked 実機テストガイド

3 分でわかる TL;DR
ローカルビルドした dist/chrome と dist/firefox を各ブラウザに「開発者モードで」読み込む。テストは 5 シナリオ（空状態 / スターター取込 / 冪等性 / teach-mode / 使い方ページ遷移）。teach-mode は専用の新規タブと Continuous mode を使い、複数 metric とキャンセル時の全破棄まで確認する。
2026-07-19 追記: スターターは resources/starter.json（GitHub raw）。未 push だと 404 → ローカル JSON 貼り付けで代替。Claude の登録 URL は https://claude.ai/new#settings/usage。

1. 前提の確認

テストに入る前に以下を確認する。環境が変わっているときは念のため。

確認事項確認方法状態
ビルド成果物の存在C:\dev\github\public\many-ai-usage\dist\chrome と dist\firefox に manifest.json があること。無ければ pnpm run build:chrome / pnpm run build:firefox を実行。ソースを直したら必ず再 build → 拡張の再読み込み☐
typecheck / testpnpm run typecheck と pnpm test が緑☐
スターター JSON（remote）https://raw.githubusercontent.com/ishizakahiroshi/many-ai-usage/main/resources/starter.json が 200 か。404 なら resources/ が main に未 push → 下の「貼り付け import」で代替☐
スターター JSON（ローカル）resources/starter.json が存在する（貼り付け用）。Claude の url が https://claude.ai/new#settings/usage であること☐

remote が 200 なら「スターターを取り込む ▸」だけで T2 が通る。404 でも貼り付け import で dogfood は続行できる。

2. Chrome への unpacked インストール

  1. アドレスバーに chrome://extensions/ と入れて開く
  2. 右上の「デベロッパー モード」トグルを ON（青くなる）
  3. 左上に現れる「パッケージ化されていない拡張機能を読み込む」を押す
  4. ファイル選択ダイアログで C:\dev\github\public\many-ai-usage\dist\chrome フォルダに入って、そのまま「フォルダーの選択」ボタンを押す
    迷いポイント: ここは「フォルダの中に入って、フォルダ自体を選ぶ」動作。
    ❌ リポルート C:\dev\github\public\many-ai-usage を選ぶ → manifest 見つからず失敗
    ❌ dist\chrome の中の manifest.json を選ぶ（Firefox の作法と混同）→ ダイアログが閉じない
    ✅ dist\chrome フォルダを選択状態で「フォルダーの選択」
  5. 拡張カードが出て「many-ai-usage v0.1.0」が表示される。右上のパズルアイコン（拡張メニュー）に M か many-ai-usage のアイコンが並ぶ
  6. 拡張アイコンを右クリック →「拡張機能を固定」でツールバーに常駐させると以降が楽

3. Firefox への unpacked インストール

  1. アドレスバーに about:debugging#/runtime/this-firefox と入れて開く（通常版・Developer Edition どちらでも可）
  2. 「一時的なアドオンを読み込む…」ボタンを押す
  3. ファイル選択ダイアログで C:\dev\github\public\many-ai-usage\dist\firefox\manifest.json を選ぶ（フォルダではなくファイルを選ぶ）
    Chrome との違い: Firefox は manifest.json ファイルそのものを指定する。
  4. 拡張カードが出て「many-ai-usage 0.1.0」が表示され、バックグラウンドが「実行中」なら成功
  5. 再 build 後はこの画面の「再読み込み」を押す（または削除して再読込）
  6. ツールバーにアイコンが無ければ パズル（拡張）メニュー → many-ai-usage をピン留め
注意: Firefox の「一時的なアドオン」はブラウザを閉じると消える。テストごとに再読込が要る。恒常的に使うなら AMO 提出後の署名付きパッケージを待つこと。
プライベートウィンドウ: Ctrl+Shift+P。拡張の有無切り分けに使えるが、ログイン Cookie は通常ウィンドウと共有されない。teach 本番はログイン済みの通常ウィンドウ推奨。
「Firefox 非対応？」と疑う前に: 各 AI サイト自体がその Firefox でログイン・usage 表示できるか先に確認する。拡張は登録 URL を 1 回開くだけで、サイトのログイン壁・upgrade リダイレクトはサイト仕様。

3.5 サービス別・登録 URL（dogfood 2026-07-19）

スターター取込後・手動登録時の 開く URL。ここがズレると teach 以前に usage 数字が出ない。

サービス推奨登録 URLメモ
Claudehttps://claude.ai/new#settings/usage/settings/usage は upgrade 等に寄りやすい。hash SPA のこちらを使う（starter.json も同値に修正済み）
Codex CLIhttps://chatgpt.com/codex/cloud/settings/analytics#usageChatGPT ログイン必須
Grokhttps://grok.com/?_s=usage未ログインだと薄いアカウント UI になる（拡張のループではない）

teach 前に「画面に % や数値が目で見える」ことを確認してから Track する。

4. テストシナリオ 5 種

順番通り実行するのが推奨。T1 が終わってから T2 → T3 → T4 → T5。各シナリオの「成功基準」を満たしたら次へ。

T1空状態フロー

ねらい: 初回起動時にスターター導線（Try samples / スターター取込）が出るか
  1. 拡張アイコンをクリックして popup を開く
  2. popup 全体を確認。設定（options）も開けること
providers が既に登録済みなら: Chrome は chrome://extensions/ → many-ai-usage「詳細」→「拡張機能のデータを消去」。Firefox は一時アドオンを削除して再読込するか、storage を空にする。空状態で T1 をやり直す。
成功基準:
  • ヘッダ「many-ai-usage v0.1.0」
  • 空メッセージ（スターター取込 or 自分の usage 追加を促す文言）
  • スターター／Try samples 系ボタンと「使い方を見る」系リンクが見える
  • ⚙ 設定で options タブが開く

T2スターター取込（Remote または貼り付け）

ねらい: 明示同意 → starter.json から最大 6 サービス → storage に反映。remote 404 時はローカル貼り付けで同等結果

経路 A: remote（main に resources があるとき）

  1. options を開く（popup の ⚙）
  2. 左サイド下部（「不具合を報告」の上）の「スターターを取り込む ▸」を押す。空状態では上部 onboarding 枠にも同じボタンがある
  3. 確認ダイアログの URL が https://raw.githubusercontent.com/ishizakahiroshi/many-ai-usage/main/resources/starter.json であること
  4. 取込を実行し、数秒待つ

経路 B: 貼り付け（remote が 404 のとき）

  1. PowerShell: Get-Content -Raw C:\dev\github\public\many-ai-usage\resources\starter.json | Set-Clipboard
  2. options の「JSON を貼り付けてインポート」を開き、貼り付け → インポート
成功基準:
  • 左サイドに 6 タイル相当: Claude / Codex CLI / Grok / GitHub Copilot / Cursor / Ollama
  • Claude の URL が https://claude.ai/new#settings/usage（options で確認）
  • 未 teach のサービスはタイル／要対応扱い（正常）
失敗パターン: 「Starter pack request failed (404)」→ resources/ が GitHub main に無い。経路 B で続行するか、commit+push 後に経路 A を再試行。

T3スターター取込の冪等性

ねらい: 2 回目の取込で既存 provider を壊さず、不要な重複を増やさないこと
  1. T2 完了後、再度「スターターを取り込む」または options.html?importStarter=1 / ?trySamples=1 でダイアログを出す
  2. 取込を実行する
成功基準:
  • タイル数がむやみに倍増しない（既登録メッセージ or 上書き確認の仕様どおり）
  • 既に teach 済みの selector / 表示が消えない

T4teach-mode Continuous flow

ねらい: 専用の新規タブで複数 metric を連続 teach し、完了時にまとめて保存されることを確認
  1. https://chatgpt.com/codex/cloud/settings/analytics#usage を同じウィンドウで 2 タブ開き、ChatGPT にログインしておく
  2. 拡張の options タブに戻る（chrome://extensions/ → many-ai-usage →「オプション」or popup の ⚙ options から）
  3. 左サイドバーで Codex CLI を選択
  4. 右パネル下部の 「＋ Track this element」 ボタンを押す
  5. Chrome が Codex usage を3 個目の新規タブとして開き、そのタブだけに picker を表示する
  6. ページ上の数値要素にマウスを重ねると薄オレンジのハイライトが出る
  7. 「X% 残り」の数字部分をクリックし、右上 panel の Saved が 1 になる
    日付の年号へマウスを重ねても「2026%」にならず、リセット文脈の数字は候補にならないことを確認する
  8. 続けて「残りのクレジット」をクリックし、Saved が 2 になる
  9. panel の Rename で片方の名前を変更し、一覧へ反映されることを確認する
  10. Done and return を押し、新規 picker タブが閉じて options タブへ戻る
  11. Codex CLI の「Tracked elements」に taught metric が 2 つ表示される
成功基準:
  • 既存 2 タブではなく、新規の 3 個目のタブだけに picker が出る
  • percentage と credits の 2 metric を 1 セッションで保存できる
  • tooltip が年号を usage 値として表示しない
  • options の「Tracked elements」と popup に 2 metric が反映される
キャンセル確認: もう一度 Track this element を開始して 1 metric を選び、Esc を押す。options に戻った後、Tracked elements が増えていなければ全破棄も成功。
Claude で試す場合: 登録 URL を https://claude.ai/new#settings/usage にし、ログイン済みで % が見えてから Track。セッション／週間など複数 metric を Continuous で teach できる。
Grok でマーカーがおかしいとき: 枠がサイドバーまで伸びる・ツールチップが変な数値（例: 4.5）→ usage タブで Ctrl+Shift+R（スーパーリロード）してから Track し直す。SPA + content script の半端状態で起きやすい。

T5使い方ページ遷移

ねらい: popup / options の「使い方を見る →」リンクが正しく github.io に飛ぶ
  1. popup を開く（provider 空状態でも、6 個登録済み状態でも良い）
  2. 空状態なら「使い方を見る →」ボタン、登録済みなら options の「Try samples」onboarding 内のリンクを押す
成功基準:
  • 新規タブが開いて https://ishizakahiroshi.com/articles/many-ai-usage/usage.html が表示される
  • ページが 404 でない
  • 使い方ページの各セクションが崩れずに表示される

6. 不具合が出た時のトラブルシュート

よくある症状（2026-07-19 dogfood）

症状まず疑うこと対処
Starter pack request failed (404)resources/starter.json が GitHub main に無い貼り付け import、または resources を commit+push
Claude が upgrade / 真っ黒 / 使い物にならない登録 URL が /settings/usageoptions で https://claude.ai/new#settings/usage に変更。ログイン済みか確認
Grok が薄いアカウント UI だけ未ログインサインイン後に ?_s=usage を開き直す（拡張の無限リダイレクトではない）
teach マーカーがサイドバーまで伸びる・変な数値Grok SPA の半端な DOM / inject 状態usage タブをスーパーリロード（Ctrl+Shift+R）して再 Track
Firefox で拡張が消えた一時アドオンの仕様manifest.json を再度読み込む
ソースを直したのに挙動が古いdist 未 build / 拡張未リロードpnpm run build:firefox（or chrome）→ 再読み込み

拡張の Service Worker のログを見る（Chrome）

  1. chrome://extensions/ を開く
  2. many-ai-usage カードの中で「Service worker」というリンク（青字）をクリック
  3. DevTools が開くので Console タブでエラーを確認
Service worker が非アクティブと表示される場合、拡張アイコンを 1 度クリックしてから再度リンクを探すと表示される

Firefox 側の Console

  1. about:debugging#/runtime/this-firefox を開く
  2. many-ai-usage の「調査」ボタン
  3. DevTools の Console タブでエラーを確認
