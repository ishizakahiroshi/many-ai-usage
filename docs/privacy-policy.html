many-ai-usage privacy policy

many-ai-usage reads visible usage information from usage pages that the user registers and opens in their own browser. Parsing occurs locally in the browser.

The extension does not send page HTML, cookies, authentication tokens, account identifiers, or normalized usage data to an external server. It does not operate a server or provide cloud synchronization.

The extension requests host access only for registered usage-page origins. It performs read-only capture and does not submit forms, send messages, or change provider settings.

Stored data is limited to provider settings, taught selectors/fingerprints, normalized snapshots, and diagnostic summaries in the browser's extension storage. Users can remove provider entries from the settings page.
